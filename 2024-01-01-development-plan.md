# 个人综合应用系统 - 功能说明文档与开发规划

---

## 一、项目概述

### 1.1 项目定位
本项目是一个面向个人使用的综合性数据管理应用，整合音声、小说、TWICE Wiki、拼豆库存、OneDrive云盘等多个功能模块，提供统一的数据管理平台。

### 1.2 技术架构
| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | Vue 3 + Vite + TypeScript | 现代化前端框架 |
| UI组件 | Ant Design Vue | 企业级组件库 |
| 移动端 | PWA | 一次开发，多端适配 |
| 后端 | Django 4 + DRF | 成熟稳定的Python框架 |
| 数据库 | MySQL (sqlpub) | 云端托管关系型数据库 |
| 媒体存储 | OneDrive + Alist | 云端媒体文件管理 |
| 认证 | JWT Token | 无状态认证机制 |
| 部署 | Docker Compose + Nginx | 容器化部署 |

### 1.3 核心价值
- 统一管理个人数字资产（音声、小说、收藏）
- 支持Web和移动端访问
- 数据云端同步，安全可靠
- 界面精致，操作便捷

## 二、功能模块说明

### 2.1 音声管理模块

#### 2.1.1 功能概述
管理个人音声收藏，支持音声信息管理、播放、标签分类等功能。

#### 2.1.2 核心功能
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 音声列表 | 分页展示音声列表，支持搜索、筛选、排序 | 高 |
| 音声详情 | 显示音声完整信息，包含封面、简介、声优等 | 高 |
| 音声播放 | 内置音频播放器，支持进度记忆 | 高 |
| 标签管理 | 为音声添加/移除标签，支持分类筛选 | 高 |
| 收藏评分 | 收藏音声、1-5星评分 | 中 |
| 爬虫抓取 | 从指定网站抓取音声元信息 | 中 |
| 云盘同步 | 扫描OneDrive目录自动创建记录 | 中 |

#### 2.1.3 关联数据模型
- **Audio（音声表）**：核心数据，存储音声基本信息
- **Circle（社团表）**：音声制作社团信息
- **VoiceActor（声优表）**：声优详细信息
- **AudioVoiceActor（关联表）**：音声与声优的多对多关系
- **AudioTag（标签表）**：音声标签分类

---

### 2.2 小说管理模块

#### 2.2.1 功能概述
管理个人小说收藏，支持在线阅读、进度追踪、云盘备份等功能。

#### 2.2.2 核心功能
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 小说列表 | 分页展示小说，支持状态标记（未读/阅读中/已读完） | 高 |
| 小说详情 | 显示小说信息、作者、平台、简介等 | 高 |
| 在线阅读 | 章节分页显示，进度自动保存 | 高 |
| 阅读设置 | 字体大小、主题切换（白天/夜间/护眼） | 高 |
| 云盘备份 | 上传至OneDrive，自动分类 | 中 |
| 爬虫抓取 | 从指定平台抓取小说信息 | 中 |

#### 2.2.3 关联数据模型
- **Novel（小说表）**：核心数据，包含标题、作者、平台、简介、阅读次数等
- **Author（作者表）**：小说作者信息
- **Platform（平台表）**：小说发布平台信息
- **NovelTag（标签表）**：小说标签分类

---

### 2.3 TWICE Wiki模块

#### 2.3.1 功能概述
管理TWICE相关信息，包括成员、专辑、歌曲等数据。

#### 2.3.2 核心功能
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 成员管理 | 成员列表展示、详情页、社交媒体链接 | 高 |
| 专辑管理 | 专辑列表、详情（含歌曲列表）、封面展示 | 高 |
| 歌曲管理 | 歌曲列表、歌词展示、时长统计 | 中 |
| 演出记录 | 演唱会列表、打歌舞台记录 | 低 |

#### 2.3.3 关联数据模型
- **Member（成员表）**：TWICE成员信息
- **Album（专辑表）**：专辑信息
- **Song（歌曲表）**：歌曲信息
- **AlbumMembers（关联表）**：专辑与成员的关系

---

### 2.4 拼豆库存模块

#### 2.4.1 功能概述
管理拼豆库存，支持库存查询、出入库记录、低库存提醒等功能。

#### 2.4.2 核心功能
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 库存列表 | 颜色列表展示，品牌筛选，库存数量编辑 | 高 |
| 库存操作 | 入库记录、出库记录、库存调整 | 高 |
| 低库存提醒 | 设置阈值，预警显示 | 高 |
| 统计分析 | 库存总量、品牌分类统计、变化趋势 | 中 |
| 批量导入 | 支持批量导入导出库存数据 | 低 |

#### 2.4.3 关联数据模型
- **Bead（拼豆表）**：拼豆库存信息
- **StockOperation（操作记录表）**：库存操作历史

---

### 2.5 云盘管理模块

#### 2.5.1 功能概述
管理OneDrive云盘文件，支持文件浏览、上传、下载等操作。

#### 2.5.2 核心功能
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 文件浏览 | 目录树导航、文件列表、缩略图预览 | 高 |
| 文件操作 | 上传、下载、新建文件夹、重命名、删除 | 高 |
| Alist集成 | 统一管理多个存储源 | 中 |
| 文件同步 | 扫描目录自动创建数据库记录 | 中 |

#### 2.5.3 关联数据模型
- **FileItem（文件表）**：云盘文件记录

## 六、开发规范

### 6.1 代码规范
- Python：遵循PEP 8规范
- TypeScript：遵循ESLint标准规则
- 命名：使用snake_case（Python）/ camelCase（TypeScript）

### 6.2 API规范
- 统一响应格式（code、message、data、timestamp）
- 错误响应包含errors字段
- 使用RESTful风格命名

### 6.3 分支管理
- `main`：生产分支
- `develop`：开发分支
- `feature/xxx`：功能分支
- `bugfix/xxx`：修复分支
